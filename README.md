# Partner OS — AI CRM для Affiliate Manager

MVP. Цель продукта: менеджер никогда не забывает про перспективного партнёра
и всегда понимает, что делать дальше.

## Стек

Next.js 14 (App Router) · TypeScript · Tailwind CSS · Supabase (Auth + Postgres) · Vercel

## Статус MVP

- [x] Структура проекта
- [x] Схема БД Supabase (`supabase/schema.sql`)
- [x] Экран 1 — Dashboard (реальные данные из Supabase)
- [x] Вход по email + паролю (Supabase Auth), защита экранов через middleware
- [ ] Экран 2 — Partners (список, фильтры)
- [ ] Экран 3 — Карточка партнёра

## Запуск локально

```bash
npm install
cp .env.local.example .env.local   # заполнить своими Supabase-ключами
npm run dev
```

## База данных

1. Создать проект в Supabase.
2. Выполнить `supabase/schema.sql` в SQL Editor.
3. Скопировать `Project URL` и `anon key` в `.env.local`.

Схема включает:
- `partners`, `projects`, `tasks`, `timeline_events`, `ai_memory_facts`
- RLS-политики: каждый менеджер видит только свои данные
- Триггеры, которые автоматически пишут в `timeline_events` при создании
  задачи, завершении задачи и создании проекта — это то самое требование
  брифа «Timeline должен обновляться автоматически», реализованное на уровне
  БД, а не на дисциплине фронтенда.

## Дизайн-система

Тёмная, "рабочая" палитра в духе операционного дашборда, а не маркетингового
лендинга — акцент на быстром принятии решений:

- `ink` / `panel` / `panel-raised` / `line` — фон и поверхности
- `paper` / `fog` — основной и вторичный текст
- `signal` (мятно-тиловый) — AI-рекомендации и основные действия
- `ember` — требует внимания / средний приоритет
- `risk` — риск потерять партнёра / высокий приоритет
- `drift` — нейтральная вторичная информация

Шрифты: Space Grotesk (заголовки), Inter (текст), IBM Plex Mono (данные,
таймстемпы, голос AI — подчёркивает, что это "показания приборов", а не
просто текст).
